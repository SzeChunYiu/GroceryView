# Security Policy

## Supported versions

GroceryView is under active development. Security fixes are applied to the `main` branch and released through the normal deployment pipeline. Older commits, preview builds, and unmaintained forks are not supported.

## Reporting a vulnerability

Please report suspected vulnerabilities privately by opening a GitHub security advisory or by contacting the maintainers through the repository owner's published GitHub contact methods. Do not disclose the issue publicly until the maintainers have had time to investigate and prepare a fix.

When reporting, include:

- a clear description of the vulnerability and affected area;
- steps to reproduce or a proof of concept, if available;
- any known impact to grocery prices, user data, authentication, or availability;
- your preferred contact information for follow-up.

## Disclosure policy

We will acknowledge reports as soon as practical, investigate the impact, and coordinate remediation before public disclosure. If the report is valid, we will publish an appropriate fix and may document the issue in release notes or a security advisory. GroceryView does not currently operate a paid bug bounty program.
