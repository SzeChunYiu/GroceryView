# Preem Sweden fuel connector notes

- **Connector id:** `preem-se`
- **Domain:** fuel prices, Sweden
- **Status:** research-only / blocked for automated price ingestion
- **Last verified:** 2026-05-24

## Source posture

Preem does not currently expose a public, machine-readable consumer fuel-price
feed that GroceryView can ingest as station-level observations. The current
Preem consumer price page states that Preem does not publish recommended
approximate prices for petrol and diesel for private customers, and that the
current price is found on the price sign at the nearest Preem station. The older
archived private fuel-price page similarly says recommended fuel prices are no
longer published on the website.

Because of that source posture, a production `preem-se` connector must not infer
station prices from national averages, archived pages, card-discount pages, or
third-party crowdsourced sites. Those sources can be useful for manual research,
but they are not suitable as authoritative Preem station observations unless a
separate source agreement or explicit public API is approved.

## Fields a future approved connector should extract

If Preem later publishes an approved station API, data feed, or signed partner
export, the connector should emit one fuel observation per station and fuel grade
with these fields:

- `stationId` and station display name
- street address, locality, coordinates, and optional opening-hours metadata
- fuel grade (`95`, `98`, `diesel`, `hvo100`, `e85`, or a mapped extension)
- current price per litre, currency `SEK`, and VAT/tax inclusion status when
  stated
- source URL or feed identifier used for the exact observation
- `retrievedAt` timestamp generated by the ingestion run
- source-effective timestamp if Preem publishes one
- parser/feed version and immutable raw snapshot hash

## Known quirks and edge cases

- Preem distinguishes private consumer pump prices from business/list-price and
  card-discount contexts; these must not be mixed.
- Station signs may differ by branch and time of day, so national list prices or
  averages are not valid per-station observations.
- Archived pages are useful for explaining why ingestion is blocked, but should
  not be scraped for current prices.
- Fuel availability differs by station. A connector should only emit grades that
  the source explicitly lists for that station.
- Third-party pages may contain user-submitted values. If used in a separate
  crowdsourced lane, rows need reporter/source provenance and freshness checks.

## Current recommendation

Keep `preem-se` out of the automated daily fuel connector set until an approved
Preem source can provide current station-level pump prices. For now, GroceryView
can document the source gap and continue to model fuel ingestion through sources
that expose explicit, retrievable price evidence.
