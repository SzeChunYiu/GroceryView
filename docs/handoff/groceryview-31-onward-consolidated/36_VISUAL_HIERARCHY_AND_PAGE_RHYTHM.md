# 36 — Visual Hierarchy and Page Rhythm

## Rule

Every page needs one dominant thing.

Bad:

```text
10 cards all same size and color.
```

Good:

```text
1 hero / primary visual
2–3 supporting panels
lower-priority detail sections
```

## Page rhythm

Use this vertical structure:

```text
1. Header / hero
2. Primary task area
3. Insight/result area
4. Supporting panels
5. Evidence/details
6. Ads where allowed
7. Footer
```

## Above the fold

Home:

```text
Hero + search + primary CTAs
```

Search:

```text
Search bar + filters + first results
```

Market:

```text
KPI row + main chart
```

Product:

```text
Image + price answer + where to buy
```

Map:

```text
Layer controls + map
```

## Reduce noise

Move these down or hide behind details:

```text
parser version
source table names
internal algorithm names
debug/performance implementation details
long caveats
raw source URLs
```

Keep visible:

```text
what this means
price
trend
confidence
freshness
next action
```

## Density levels

Home:

```text
low density
```

Search:

```text
medium density
```

Market:

```text
high density but structured
```

Product:

```text
medium-high density
```

Data sources/admin:

```text
high density allowed
```
