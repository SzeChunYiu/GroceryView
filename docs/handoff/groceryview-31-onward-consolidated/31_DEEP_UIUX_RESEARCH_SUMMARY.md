# 31 — Deep UI/UX Research Summary

## Diagnosis

GroceryView has a strong product idea, but the interface can feel unattractive when it lacks a unified visual system.

The main issues to solve are:

```text
1. Too many panels look equally important.
2. Technical copy competes with user-facing meaning.
3. Product cards, data cards, charts, and evidence panels do not share one polished design language.
4. Pages have content, but weak visual hierarchy.
5. Charts exist, but not every chart has a clear story.
6. Navigation and panels still feel like features were added one by one.
7. There is not yet a strong brand identity.
```

## Research principles applied

```text
NN/G:
- visibility of system status
- match the user's language
- consistency and standards
- recognition rather than recall
- aesthetic and minimalist design

W3C:
- charts and complex visuals need accessible descriptions and fallbacks

Google AdSense:
- ads must be distinguishable from content and placed to avoid accidental clicks

Tableau / Power BI:
- dashboards need a clear purpose, hierarchy, primary visual, filters, and drillthrough
```

## New GroceryView design thesis

```text
GroceryView should look like:
Nordic fintech + grocery intelligence + trustworthy public-data dashboard.
```

Not:

```text
generic green website
university data demo
admin dashboard
random Tailwind cards
supermarket flyer clone
```

## Visual inspiration categories

Do not copy any website directly. Learn patterns:

```text
Google Search:
fast, familiar, result-first, relevance-first

Google Maps:
layered spatial exploration and selected detail panel

Power BI / Tableau:
filters, KPI cards, drillthrough, cross-filtering, dashboard hierarchy

Bloomberg / TradingView:
dense market information, but with strong hierarchy and charts

Klarna / modern fintech:
soft cards, strong whitespace, simple financial clarity

GOV.UK:
plain language, service clarity, task-focused flow

Carbon / Atlassian:
design tokens, semantic colors, states, reusable component libraries
```
