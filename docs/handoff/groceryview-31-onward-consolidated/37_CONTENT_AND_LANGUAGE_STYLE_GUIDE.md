# 37 — Content and Language Style Guide

## Voice

```text
Plain
Calm
Helpful
Evidence-aware
Not hypey
Not technical-first
```

## Sentence length

Prefer:

```text
8–18 words.
```

Avoid long multi-clause explanations in cards.

## Replace

```text
"verified ingestion pipelines"
→ "verified price sources"

"server-side cursor pagination"
→ "showing results"

"dated OpenPrices tape"
→ "price history from observations"

"domain=fuel observations"
→ "verified fuel price rows"

"not inferred"
→ "not guessed"
```

## Use strong plain headings

Good:

```text
Is this deal actually good?
Where can I buy it?
What changed this week?
Why we trust this price
Not enough data yet
```

Bad:

```text
source tape
terminal view
domain foundation
matched observation guardrail
```

## Evidence copy formula

```text
Source: [source]
Last observed: [date]
Confidence: [level]
Rows: [count]
Limitation: [if any]
```

## CTA copy

Use action verbs:

```text
Open product
Compare stores
Browse category
View market trend
Set price alert
Open map
Check evidence
```

Avoid:

```text
Learn more
View
Click here
```

unless context is obvious.
