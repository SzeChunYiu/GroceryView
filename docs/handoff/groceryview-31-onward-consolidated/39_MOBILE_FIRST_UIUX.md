# 39 — Mobile-First UI/UX

## Mobile principle

Most shoppers will use GroceryView on mobile.

Mobile must optimize for:

```text
search
scan/list
quick deal judgement
nearby store/map
open product
watchlist
```

## Mobile nav

```text
Home
Search
Deals
Browse
Map
More
```

More:

```text
Market
Watchlist
Pharmacy
Fuel
Account
Data sources
```

## Mobile layout

Use one-column cards.

Avoid:

```text
wide tables without card fallback
tiny chart labels
horizontal overflow as primary interaction
dense side rails
```

## Mobile dashboard behavior

Market filters become:

```text
filter drawer
sticky selected-filter summary
```

Category table becomes cards:

```text
Category
Weekly / 3M / 1Y
Trend
Cheapest chain
Browse
```

Product quote table becomes cards.

Map:

```text
map top
bottom sheet selected detail
list fallback
```

## Touch targets

Minimum:

```text
44px height for buttons and chips
comfortable spacing between ads and CTAs
```
