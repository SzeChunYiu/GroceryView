# UI/UX Research Sources Used

These are the source categories behind the UI/UX recommendations.

## Primary sources

- Nielsen Norman Group — 10 Usability Heuristics
  - visibility of system status
  - match between system and real world
  - consistency and standards
  - recognition rather than recall
  - aesthetic and minimalist design
  - helpful error recovery

- W3C WAI — Complex Images
  - charts and graphs need accessible text descriptions and/or adjacent explanations or data tables when short alt text is not enough

- Google AdSense — Ad placement policies
  - avoid accidental clicks
  - make ads distinguishable from content
  - label ads only as Advertisement or Sponsored Links

- Tableau — Dashboard best practices
  - dashboards need clear purpose, audience, hierarchy, and carefully chosen key visuals

## Pattern references

- Power BI docs — dashboard visual interactions and drillthrough
- Google Search Central — how search works
- Algolia docs — typo tolerance and faceting
- Elasticsearch docs — BM25 / lexical retrieval
- IBM Carbon Design System — reusable components and tokens
- Atlassian Design System — semantic color roles and design tokens
- GOV.UK Design System — plain language and task-focused patterns
