## Not loyalty programs (do not show as enrollment CTAs yet)

- REMA 1000 Denmark/Norway apps: useful for shopping lists, recipes, offers, Scan Selv/Køb & Hent, and profiles, but no separate points-style loyalty enrollment found in the checked public surfaces.
- Discount chains without confirmed account-based loyalty in the current crawler scope (for example Bónus/Krónan public offer pages) should remain "offers only" until a verified enrollment URL and data policy are available.

## UI implementation notes

- Store `cost_type` separately from `cost_amount`: refundable co-op shares are not subscriptions.
- Label pharmacy programs as sensitive because linking can imply health or prescription data.
- Keep partner-offer data sharing opt-in in UI copy; several programs use partner benefits as a core feature.
- Re-check fees/terms on every country launch because co-op shares and app benefits differ by market and can change.
