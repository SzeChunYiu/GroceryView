# Products API

## `GET /api/products`

Lists GroceryView products using cursor pagination. The route is public and shares the same response envelope as `GET /api/products/search`; omit `q` to return the stable default product ordering.

- **Method:** `GET`
- **Path:** `/api/products`
- **Authentication:** none
- **Cache:** eligible for normal public HTTP/API caching when the server cache layer is enabled
- **Pagination model:** cursor-only; clients must follow `pagination.nextCursor` and must not synthesize offset page numbers

## Query parameters

Effective validation in `packages/server/src/index.ts` is equivalent to this Zod schema:

```ts
import { z } from 'zod';

const groceryViewCursor = z
  .string()
  .base64url()
  .transform((cursor, ctx) => {
    try {
      const decoded = JSON.parse(Buffer.from(cursor, 'base64url').toString('utf8'));
      if (!Number.isInteger(decoded?.offset) || decoded.offset < 0) throw new Error('bad cursor');
      return cursor;
    } catch {
      ctx.addIssue({ code: z.ZodIssueCode.custom, message: 'cursor must be a valid GroceryView cursor pagination token.' });
      return z.NEVER;
    }
  });

export const getProductsQuerySchema = z.object({
  q: z.string().optional().default(''),
  limit: z.coerce.number().int().positive().max(100).optional().default(25),
  cursor: groceryViewCursor.optional()
});
```

| Parameter | Type | Default | Notes |
| --- | --- | --- | --- |
| `q` | string | `""` | Optional search text. Terms are split on whitespace and matched case-insensitively against ticker/id, name, category, brand tier, and available chains. `GET /api/products` accepts it for parity with `/api/products/search`, but product listing callers normally omit it. |
| `limit` | positive integer, max `100` | `25` | Page size. Values above `100` are capped to `100`; non-integers and values below `1` return `400`. |
| `cursor` | GroceryView cursor token | none | Opaque base64url JSON token generated by `pagination.nextCursor`, currently encoding an internal `{ "offset": number }`. Invalid tokens return `400`. |

## Response shape

`200 OK` returns a cursor-paginated envelope:

```ts
type ProductsResponse = {
  items: Product[];
  pagination: {
    limit: number;
    cursor: string | null;
    nextCursor: string | null;
    hasMore: boolean;
    totalReturned: number;
    totalItems: number;
    totalProductCount: number;
    source: 'cursor pagination over stable product search ordering';
  };
  guardrails: string[];
};
```

`Product` is the public product summary from `@groceryview/core`/`@groceryview/api` search data. Current fields include:

```ts
type Product = {
  id: string;
  ticker: string;
  name: string;
  category: string;
  brandTier: string;
  availableChains: string[];
};
```

Deployments may include additional product fields as the public catalog model evolves; clients should ignore unknown fields.

## Error codes

| Status | Body | When |
| --- | --- | --- |
| `400` | `{ "error": "limit must be a positive integer." }` | `limit` is non-numeric, non-integer, or less than `1`. |
| `400` | `{ "error": "cursor must be a valid GroceryView cursor pagination token." }` | `cursor` is not an opaque GroceryView cursor or decodes to an invalid offset. |
| `404` | `{ "error": "Route not found: <METHOD> /api/products" }` | A non-`GET` method is sent to the route or the route path is otherwise not registered for that method. |
| `500` | `{ "error": "Internal server error." }` | Unexpected server failure. |

## Rate limit

No per-route product-listing throttle is enforced in the current server handler. Treat the endpoint as a public catalog route and keep clients conservative: cache responses, debounce search input, and follow `nextCursor` instead of parallel-scanning many cursors.
